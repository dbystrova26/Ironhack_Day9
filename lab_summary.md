This lab's main challenges were keeping environment variable names consistent, handling live API failures separately from code errors, and choosing model names that the available API keys could actually access. I solved these by centralizing configuration in `config.py`, adding clear validation and rate-limit logic, mocking external calls in unit tests, and making model names configurable through `.env`. I learned that multi-provider apps need explicit fallback, cost tracking, and reliable error messages to be usable, and future improvements could include storing results, adding a small web UI, using fully async API clients, and improving retry handling for NewsAPI and LLM calls.
